⚙️ Configuration — Set up AI providers, analysis profiles, and system settings. Switch sub-tabs to see context-specific help.
  • AI Settings — Global API Keys (admin) + per-project provider/model for 3 use-cases (Document Analysis, AI Chat, Case Intelligence). Fallback chain: per-project → global → system default.
  • AI Usage — Daily cost + call volume chart. Hover a bar for exact values.
  • Profiles — Rules files that define how the AI tags and classifies documents. Activate staging profiles to use them.
  • Vector Store — Manages the embedded document index used for AI Chat. Clear it if documents seem stale.
  • Notifications (SMTP) (admin) — Configure outgoing email for welcome messages, CI budget alerts, and CI completion emails.
  • Users (admin) — Add, edit, or remove user accounts. Three roles: Admin / Advanced / Basic.

🤖 AI Settings — Provider and model configuration.
  • Global API Keys (admin only) — OpenAI and Anthropic keys stored in /app/data/ai_config.json. Click Test next to each key to verify before saving.
  • Per-project AI Configuration — Three use-cases per project: Document Analysis / AI Chat / Case Intelligence. Each has a Primary and a Fallback provider+model.
  • ⇉ All — copies that row's settings to all three use-cases in this project.
  • 📋 Copy (admin) — copies one project's full config to another project.
  • Fallback chain: per-project override → global key → system default.
  • Admin sees all-projects dropdown; Advanced users see only their own project.

💰 AI Usage — Daily token and cost history.
  • Blue bars — daily estimated cost (USD).
  • Amber dashed line — daily API call volume (right axis).
  • Hover a bar to see exact date, cost, and call count for that day.
  • Use the days selector to widen or narrow the date range.

📄 Profiles — Document-type analysis rules.
  • Active profiles are loaded from /app/profiles/active/ and define extraction rules, risk weights, and tagging for each document type.
  • Staging profiles are auto-generated when a document matches no active profile. Review and activate the ones you want to keep.
  • Activate / Activate All — move staging profiles to active.
  • Delete — permanently remove a staging or active profile.
📖 Profiles manual →
🗄️ Vector Store — ChromaDB document index.
  • Process Unanalyzed — queue docs not yet embedded. No LLM cost for already-processed docs.
  • Re-embed Stale — re-analyze docs modified in Paperless since their last embed.
  • Clear All — destructive — wipes the entire project's vector index. Use only when rebuilding from scratch.
  • Reconcile — removes orphaned DB records for docs deleted from Paperless.
📖 Vector Store manual →
📧 Notifications — SMTP outgoing email configuration (admin only).
  • SMTP is used for: welcome emails on user creation, bug reports, CI budget alerts, and CI run completion emails.
  • CI budget alerts fire at every 10% completion checkpoint with a projected-vs-budget status.
  • CI completion emails fire when a run finishes (success or failure).
  • CI notification recipient is set per-run in CI Setup → Notifications section (pre-filled from the user's profile email).
  • Click Send Test Email to validate SMTP — a test message is sent to your own account address.
📖 SMTP manual →
👥 Users — User account management (admin only).
  • Three roles: Admin (full access) / Advanced (CI + power features) / Basic (own sessions only). Role is set via the Edit (pencil) icon.
  • + Add User — creates a new account. A welcome email is sent automatically if SMTP is configured.
  • Deactivate (trash icon) — soft-deletes the account; chat history is preserved. Click Reactivate to restore access.
  • You cannot deactivate your own account. At least one Admin must remain active.
  • 👤 My Profile (header button, all users) — every user can update their own name, email, phone, address, job title, and password without admin help.
