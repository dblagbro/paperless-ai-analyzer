Configuration

Admin-only settings for AI providers, document profiles, vector store, SMTP, and users.

Most configuration sub-tabs are visible only to users with the Admin role.

AI Settings

The AI Settings sub-tab has two sections: Global API Keys (admin-only) and Per-Project AI Configuration.

Global API Keys

Global API Keys are visible only to Admin users.
  • OpenAI API Key — used for GPT-4o document analysis, metadata extraction, Vision OCR, and AI Chat.
  • Anthropic API Key — used for Claude document analysis, Vision OCR, and AI Chat (alternative to OpenAI).
  • Cohere API Key — required for the vector store (document embedding and semantic search). Without this, AI Chat and Search are disabled.

At least one of OpenAI or Anthropic is required for document analysis and AI Chat. Keys are stored in /app/data/ai_config.json and serve as the system-wide fallback when no per-project override is set.

Click Test next to each key to verify connectivity. The result shows whether the key is valid and the selected model is accessible.

Per-Project AI Configuration

Every project can independently configure which AI provider and model to use for each of three use-cases:

Use-caseDescription
Document AnalysisLLM used when the poller processes new documents from Paperless — metadata extraction, summaries, anomaly commentary.
AI ChatLLM used when users ask questions in the AI Chat tab.
Case IntelligenceLLM used by the CI Director, Managers, and Workers for legal document analysis.

For each use-case choose a Primary provider + model and an optional Fallback provider + model. If the primary call fails the system automatically retries with the fallback.

Fallback chain: per-project override → global API key → system default.

  • ⇉ All — copies one row's primary + fallback settings to all three use-cases in this project.
  • 📋 Copy project (admin only) — copies the entire project's AI config to another project.
Admin users see a project dropdown and can configure any project. Advanced users see only their own current project. Basic users see a read-only display.

Profiles

Profiles are YAML files that define document-type-specific analysis rules. Each profile specifies:

  • Keywords and patterns that identify the document type (e.g., "bank statement", "check register").
  • Which fields to extract (account numbers, balances, transaction details).
  • Custom risk scoring weights.

Active profiles

Active profiles are loaded from /app/profiles/active/. They appear in the Active section of the Profiles tab with their match count. You can:

  • View — expand to see the YAML content.
  • Rename — change the profile filename.
  • Delete — permanently remove the profile.

Staging profiles

When a document doesn't match any active profile, the analyzer auto-generates a staging profile suggestion in /app/profiles/staging/. Review each staging profile and either:

  • Activate — move it to active profiles so future documents of that type use it.
  • Activate All — activate every staging profile at once.
  • Delete — discard if the suggestion is not useful.

Vector Store

Controls for the ChromaDB vector store that powers AI Chat and Search.

ControlEffect
RefreshReload current document count and storage stats.
Process UnanalyzedFind and queue all Paperless documents not yet in the analyzer DB. No LLM calls for already-processed docs.
Re-embed StaleRe-analyze documents whose Paperless modification timestamp is newer than their last embed.
Clear AllDelete the entire vector index for the current project. Destructive — use only when rebuilding from scratch.

AI Usage & Cost

The AI Usage sub-tab shows a daily bar chart of LLM token consumption and estimated API cost, drawn from the persistent usage tracker database.

  • Blue bars — daily estimated cost in USD.
  • Amber dashed line — daily API call volume (right axis).

The chart renders in-browser using the Canvas 2D API. Hover over a bar to see the exact values for that day. For detailed per-call history and provider breakdowns, see the LLM Usage & Cost documentation page.

SMTP / Notifications

Configure outgoing email for user welcome messages, bug report submissions, and Case Intelligence run notifications.

FieldDescription
SMTP HostMail server hostname (e.g., smtp.gmail.com)
SMTP Port587 (STARTTLS) or 465 (SSL)
Username / PasswordSMTP authentication credentials
From AddressThe email address shown in the From header
TLSEnable STARTTLS (recommended)

Click Send Test Email to verify the connection before saving. A test message is sent to your own email address (configured on your user account).

Case Intelligence email notifications

When SMTP is configured, CI runs send two types of email notifications:

  • Budget alerts — sent at every 10% completion checkpoint with the status (under budget / on track / over budget), cost so far, and projected total vs. the run budget.
  • Run completion — sent when the full analysis finishes, with a findings summary.

Both types can be toggled per-run in Case Intelligence → Setup → Notifications. The recipient email is pre-filled from your user profile but can be changed before starting the run.
