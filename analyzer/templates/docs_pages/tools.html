Debug & Tools

Live logs, manual reprocess controls, and diagnostic utilities. Admin access recommended for most operations.

System Health

The Health sub-tab shows real-time status of every system component. Each card refreshes every 30 seconds automatically while the tab is open. Click Refresh to force an immediate re-check.

ComponentWhat it checks
Paperless APIHTTP connectivity to the Paperless-ngx instance; queries /api/documents/ to confirm authentication and reachability.
ChromaDBVector store connection; verifies the current project's collection exists and is queryable.
LLMAt least one AI provider key is configured and a test completion call succeeds.
Analyzer LoopConfirms the background polling thread is alive and ran within the last 5 minutes.
PostgreSQLVerifies the Paperless database is reachable from inside the analyzer container.
RedisChecks that the Paperless Redis broker responds to PING.

The six colored dots in the dashboard header bar mirror these checks and auto-update every 30 seconds regardless of which tab is open.

Container Management

The Containers sub-tab lists all managed Docker containers — the core analyzer stack plus any provisioned per-project Paperless instances. Requires the Docker socket mounted at /var/run/docker.sock.

  • Restart Admin — gracefully restarts the selected container. Use this after code or config changes applied via volume mount (no image rebuild needed).
  • View Logs — opens a live log drawer showing the last 200 lines from that container. Useful for diagnosing startup failures or integration errors in per-project Paperless instances.
  • Status indicator — shows running, exited, or restarting with the container uptime.

The container list auto-refreshes every 60 seconds while the Containers sub-tab is visible.

Docker socket required. If the socket is unavailable, a warning banner is shown and all container controls are disabled. Health and other sub-tabs are not affected.

Reconcile

The Reconcile sub-tab compares the analyzer's records against Paperless-ngx and ChromaDB, then reports and fixes discrepancies. It is a safe, read-first operation.

  • Documents in Paperless that are missing from processed_documents are queued for analysis.
  • Embeddings in ChromaDB for documents that no longer exist in Paperless are removed from the vector store.
  • Orphaned project:slug tags for deleted projects are logged for manual review.

Click Reconcile Now. The result panel shows added, removed, and unchanged record counts. Nothing is deleted without first being identified in the report.

When to use: After a Paperless database restore, after manual DB changes, or when document counts in the Overview tab don't match Paperless.

Live Logs

The log viewer shows the last 200 log lines from the analyzer process. Click Refresh Logs to reload. Logs include:

  • Document analysis events (which document was analyzed, which profile matched, what tags were applied)
  • Vector store operations (embedding success/failure, collection counts)
  • API call outcomes (LLM provider used, token counts)
  • Error and warning messages

Log lines are color-coded: white = INFO, yellow = WARNING, red = ERROR.

Reprocess All Documents

Click Reprocess All to reset the analyzer's state file and re-queue every document in Paperless for analysis from scratch. Use this when:

  • You have updated your YAML profiles and want all documents re-analyzed with the new profiles.
  • The vector store was cleared and you need to rebuild embeddings.
  • Major configuration changes (new AI provider, new prompts) warrant re-analysis of existing documents.
This is a long-running operation. For large archives (hundreds to thousands of documents), reprocessing can take hours and will consume LLM API credits for every document. Use Process Unanalyzed Only to avoid re-analyzing already-processed documents.

Reprocess a Single Document

Enter a Paperless document ID in the Single Document field and click Reprocess. This forces the analyzer to re-analyze one specific document regardless of whether it was previously processed. Useful for testing after profile changes.

Process Unanalyzed Only

Available in Configuration → Vector Store → Process Unanalyzed. This queries Paperless for all documents, compares them against the analyzer's processed_documents table, and queues only those that have never been analyzed. Already-processed documents are skipped. Use this to:

  • Catch documents that were added to Paperless while the analyzer was offline.
  • Process a gap of documents without re-running everything.

Re-embed Stale Documents

Available in Configuration → Vector Store → Re-embed Stale. The analyzer tracks each document's Paperless modification timestamp. If a document was re-OCR'd or modified in Paperless after its initial embed, this operation re-analyzes and re-embeds it with the updated content.

Poll Interval

The analyzer checks Paperless for new documents every N seconds (default: 30). You can change this via the Poll Interval control in Configuration → AI Settings. Shorter intervals mean faster pickup of new documents but slightly more API traffic to Paperless.

Vector Store tools

In Configuration → Vector Store:

  • Clear All — deletes the entire vector index for the current project. AI Chat will have no context until documents are re-embedded (via Reprocess All or the normal analysis cycle). Use with care.
  • View by Type — lists all document types present in the vector store and their counts.
  • Delete by Type — removes all documents of a specific type from the vector store (not from Paperless).
  • Delete Single Document — removes one document from the vector store by Paperless ID.
