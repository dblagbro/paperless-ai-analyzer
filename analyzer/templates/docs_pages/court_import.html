Court Document Importer

Pull entire federal or NYS court case files directly into your Paperless-ngx project library without manual downloading. Court Import is always enabled — no environment variable configuration is required.

Overview

The Court Import tab (🏛️) lets you:

  • Search federal cases via the free CourtListener/RECAP API (no account needed)
  • Fall back to PACER direct downloads when a document is not in the RECAP archive
  • Import NYS NYSCEF case files using your NY Attorney Registration credentials or a free NYSCEF Pro Se account (requires Playwright/Chromium)
  • All documents are 3-tier deduplicated so re-running an import is safe

Setting up credentials

There are two ways to enter credentials from the Court Import panel:

  • 📋 Paste credentials — the fastest path. Paste any email, Slack message, or attorney notes that contain login info. AI extracts all fields automatically, asks one follow-up question if anything is missing, and auto-fills the form. Opens the wizard directly on the AI Paste tab — no step 1 click required.
  • ⚙️ Manage — opens the 3-step setup wizard for manual entry or to test/update existing credentials.

Wizard steps (manual path)

  1. Step 1 — Select the court system: Federal (PACER + CourtListener) or NYSCEF.
  2. Step 2 — Choose ✏️ Manual or 📋 AI Paste. Enter credentials and click 🔌 Test Connection to verify before saving. For federal, PACER credentials are optional (RECAP is free). For NYSCEF, enter your NY Attorney Registration # and e-Filing password — or tick Pro Se / Party access and enter your free NYSCEF account credentials instead (see below). Password fields have a Show/Hide toggle.
  3. Step 3 — Confirmation. Credentials are AES-256-GCM encrypted at rest.

Security note: Rotating the Flask secret key (rare) invalidates all stored court passwords — you will need to re-enter them.

Federal courts (RECAP-first)

Federal court imports use a two-tier strategy:

  • CourtListener RECAP (free) — the primary source. No PACER fees for documents already archived. Supports an API token (courtlistener_api_token in extra config) to raise the 5,000 req/day anonymous limit.
  • PACER direct (fallback) — used only when a document is not in the RECAP archive and PACER credentials are configured. The connector uses session-cookie auth, a 1-second rate limit between downloads, and a realistic user-agent header.

NYSCEF (New York courts)

NYSCEF support uses Playwright headless Chromium and requires the image to be built with INCLUDE_PLAYWRIGHT=true (adds ~400 MB). Without Playwright, NYSCEF features return a clear error rather than crashing.

Set a Default County in the wizard extra config to pre-select the county dropdown on every search.

Pro Se / Party access

Attorneys use their NY Attorney Registration number as username. Parties, defendants, and plaintiffs who are not admitted attorneys need a free NYSCEF Pro Se account instead:

  1. Go to iapps.courts.state.ny.us/nyscef → Unrepresented Litigants → Create an Account.
  2. Complete registration. You will receive a username (not a 7-digit bar number) and set a password.
  3. In the credential wizard step 2, tick Pro Se / Party access. The label changes from "NY Attorney Registration #" to "NYSCEF Username" and the hint text updates accordingly.
  4. Enter your Pro Se username and password and click 🔌 Test Connection to verify.
Why Pro Se users need an account: Anonymous NYSCEF case searches are protected by CAPTCHA. Any user — attorney or Pro Se — must be logged in to search and download dockets programmatically.

Deduplication

Every document passes three dedup checks before upload:

  1. URL match — checks the court_imported_docs table for the exact source URL. The fastest check; skips previously imported docs immediately.
  2. SHA-256 hash match — if the URL is new but the file bytes are identical to a previous import, the download is discarded and the doc is marked skipped.
  3. Paperless title search — searches Paperless for a document with a similar title. Protects against duplicates that arrived through other upload paths.

Running an import

  1. Search for the case by case number or party name.
  2. Click a result to load the full docket.
  3. Use the filter bar to narrow by title or source (RECAP / PACER / NYSCEF), or select specific rows with checkboxes.
  4. Click Import All or Import Selected.
  5. Watch the live progress bar: imported · skipped · failed counters update every 2 seconds. Click Cancel to stop within one document.

Imported documents are tagged court:federal (or court:nyscef) and project:<slug> in Paperless.

Import history

The history table shows all past jobs for the current project: case number, court system, imported/skipped/failed counts, date, and status. Click ↺ Sync Again to reload the docket for a previous case — already-imported documents will be skipped by the deduplication layer.

Docker build options

  • INCLUDE_PLAYWRIGHT=false (default) — no Chromium; NYSCEF not available; full federal support works.
  • INCLUDE_PLAYWRIGHT=true — installs Playwright + Chromium; adds ~400 MB to the image; required for NYSCEF.
