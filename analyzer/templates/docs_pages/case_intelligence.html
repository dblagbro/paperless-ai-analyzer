Case Intelligence AI

Proactive, evidence-grounded legal analysis across all documents in a project — powered by a Director → Manager → Worker LLM orchestrator.

Case Intelligence is always enabled — no environment variable is required. It is available to users with an Advanced or Admin role.

What is Case Intelligence?

Case Intelligence (CI) reads every document in a project — not one at a time, but as a coordinated group — and produces a structured legal analysis covering entities, timeline, finances, contradictions, legal theories, and authority citations. The analysis is grounded in the actual documents: every finding cites a source document and page.

Use CI to:

  • Build a comprehensive case chronology from dozens of fragmented documents.
  • Identify contradictions across witness statements, bank records, and contracts.
  • Extract every person, company, and account referenced in the document set.
  • Find relevant legal authorities cited or implied in the documents.
  • Generate a litigation-ready report in scientific paper format.

Architecture

CI runs use a three-tier orchestrator:

TierCountRole
Director1 (fixed)Reads the run config and all document summaries; plans domain assignments; synthesizes the final report; writes findings back to Paperless.
ManagersAuto (1–10)Each manager owns one analysis domain. Receives a list of documents and instructions from the Director; splits work across workers; aggregates worker results into a domain report.
WorkersAuto (1–20 per manager)Each worker processes 1–3 documents with targeted extraction instructions. Returns structured JSON with mandatory provenance citations.

Six analysis domains: Entities · Timeline · Financial · Contradictions · Theories · Authorities

Two-phase execution with war room briefing: Phase 1 (Entities, Timeline, Financial) runs first. Before Phase 2 starts, the orchestrator builds a war room briefing — a compact summary of all Phase 1 findings — and injects it into every Phase 2 manager's context. This means the Contradictions, Theories, and Authorities managers all start with full awareness of what Phase 1 extracted, rather than having to re-derive facts from raw documents.

Opposing theory pass: After generating theories for your configured role, CI automatically generates theories from the opposing perspective (e.g. defense theories if your role is plaintiff). This lets you see the strongest counter-arguments before they are raised in proceedings.

Auto scaling: Managers = min(6, ceil(doc_count ÷ 20)). Workers = budget-scaled. Both are adjustable in Setup.

AI Chat integration

When a CI run completes, all findings — entities, timeline events, contradictions, theories, authorities, and disputed facts — are automatically embedded into the project's vector store. This means:

  • AI Chat queries will surface CI findings alongside document excerpts when relevant.
  • On subsequent CI runs on the same project, the Director is aware of prior CI findings and focuses analysis on new angles rather than re-extracting already-known facts.

No configuration needed — embedding happens automatically if vector search is enabled for the project.

Setup tab walkthrough

Role, Jurisdiction & Goal

  • Role — Plaintiff / Defendant / Neutral. Shapes how the Director frames the analysis and which facts it emphasizes.
  • Jurisdiction — Optional. Auto-detected from documents if left blank; or choose manually from the dropdown.
  • Goal / Case Description — Free-text description of what you want to find. Be as specific as possible: the more context the Director has, the more targeted the analysis.

Click ✨ Refine with AI next to the Goal field to open the Goal Assistant — an AI chat that asks clarifying questions about your matter and then produces a structured goal statement you can apply with one click.

Clarifying questions

After clicking Run Analysis, the Director generates 3–5 clarifying questions about the case. For each question you can:

  • Type an answer to refine the analysis direction.
  • Leave it blank to let the Director infer from the documents.
  • Check Proceed with assumptions to skip all questions and start immediately.

Findings tab

Select a completed (or in-progress) run from the dropdown. The header shows the run status, document count, total cost, and elapsed time. A Cancel button is shown while the run is active.

Sharing runs

Click 👥 Share to open the share modal. Enter another user's username and click Add. Shared users see the run in their dropdown annotated with your name. Admins always see all runs regardless of sharing.

Finding domains

Switch between the six domain tabs to explore findings. Each finding includes:

  • The extracted data (entity, event, amount, contradiction, theory, or citation).
  • A source reference: document title + page number.
  • Confidence level and analyst notes where applicable.

Cost management

Every CI run has a hard budget cap. Budget checkpoints fire at 10%, 20%, … 100% completion:

  • Under budget — projected total is below the cap; worker count may increase.
  • On track — projected total is within 10% of the cap; no change.
  • Over budget — projected total exceeds the cap; worker count is reduced. Email alert sent if notifications are on.

Cost is driven primarily by worker count and document length. Use Auto tier settings to let the system optimize within your budget, or set fixed counts if you need predictable throughput.
