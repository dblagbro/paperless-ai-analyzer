📊 Overview — At a glance, this tab shows the live status of the analyzer and key stats.
  • AI Analyzed — Total documents the AI has processed and embedded in ChromaDB.
  • Awaiting OCR — Uploaded to Paperless but OCR not yet complete (resolves automatically).
  • Awaiting AI — OCR complete in Paperless but not yet run through AI analysis. Click 🔍 Analyze Missing in Manage Projects to recover these.
  • Anomalies Detected — Documents flagged with any anomaly (balance mismatch, layout issues, etc.).
  • High Risk — Documents with critical or high-severity anomalies requiring review.
  • Recent Activity — The latest processed documents and their results.
  • Use Configuration → AI Settings to connect an AI model if no analysis is running.

📁 Manage Projects — Projects group related documents so the AI can reason across them together.
  • Click + New Project to create a project and give it a name, description, and tags.
  • Use 📦 Move Documents to reassign documents from one project to another.
  • Use 🗄️ Archive to hide completed projects without deleting them.
  • The Current Project selector in the header controls which project context the AI Chat uses.
  • Deleting a project removes it from the analyzer but does not delete documents from Paperless.
  • ⚙️ Paperless (v3.6.1) — configure a dedicated Paperless-ngx instance for total document isolation:
    • Provision tab — click ⚡ Auto-Provision Now to fully automate container startup, DB creation, nginx config, and project wiring. New projects trigger this automatically on creation. Manual snippets still available under "Manual Setup (Advanced)".
    • Connect tab — save the internal URL, API token, and public base URL. Token is AES-256 encrypted. Use Test Connection to verify before saving.
    • Migrate tab — one-click migration of all existing documents from the shared Paperless instance to the new dedicated one. Rewires ChromaDB embeddings, SQLite records, and chat history references.

                    🩺 System Health — Live status of 6 components, checked in parallel (1.8 s timeout each). Auto-refreshes every 30 s while this sub-tab is open.
                    
    
                        
  • Paperless API — Calls the Paperless health endpoint. Red here means document analysis will fail for all users.
    • 
                        
  • ChromaDB — Counts documents in the default vector store. Yellow if vector search is disabled; red if the database is unreachable or corrupt.
    • 
                        
  • LLM — Reads LLM_ENABLED and LLM_API_KEY env vars. No live API call is made — yellow means disabled, red means enabled but no key set.
    • 
                        
  • Analyzer Loop — Checks age of last completed poll. Yellow if more than 3× the poll interval has elapsed (default threshold: 90 s).
    • 
                        
  • PostgreSQL / Redis — Checks running status via the Docker socket. Yellow (not red) when the Docker socket is unavailable in this container.
    • 
                    

                    
The header dots mirror this data and update every 30 s regardless of which tab you're on. Click them to jump here.

                

                

                    🐳 Container Manager — View and control the 7 managed Docker containers without leaving the browser. Restart and log actions require Admin role.
                    
    
                        
  • Restart — Sends SIGTERM, waits up to 15 s, then SIGKILL. The container will be briefly unavailable. The list reloads 3 s after you confirm.
    • 
                        
  • View Logs — Fetches the last 100 lines with RFC 3339 timestamps from Docker. Click the same button again to close that container's log drawer.
    • 
                        
  • Grey "not found" rows are normal for containers that don't run in every environment (e.g. paperless-ai-analyzer-jacob).
    • 
                        
  • If the list shows "Docker unavailable", add /var/run/docker.sock:/var/run/docker.sock to this container's volumes in docker-compose.yml and recreate it.
    • 
                    

                    
Auto-refreshes every 60 s while this sub-tab is open.

                

                

                    🔄 Reprocess Controls — Force the analyzer to re-examine documents it has already processed.
                    
    
                        
  • Reprocess All — Clears every existing analysis result and re-queues all documents. Use this after changing profiles, AI settings, or anomaly rules. Processing runs on the next poll cycle (every 30 s) and may take minutes to hours depending on your document count.
    • 
                        
  • Reprocess Document — Re-queues a single document by its Paperless document ID. Useful for testing a profile change on one document before committing to a full reprocess.
    • 
                    

                    
⚠️ Reprocessing overwrites previous AI results and re-applies all current tags and anomaly rules.

                

                

                    🔁 Reconcile Index — Removes orphaned records from the local database and vector store after documents are deleted from Paperless.
                    
    
                        
  • When you delete a document in Paperless, the analyzer's DB record and ChromaDB embedding are not removed automatically. Over time this causes the document count to drift and wastes vector store storage.
    • 
                        
  • Reconcile checks every local record against the Paperless API and removes any that no longer exist.
    • 
                        
  • Does not re-analyze anything — safe to run at any time without triggering LLM calls or tag changes.
    • 
                        
  • If reconcile removes a large number of records, a Reprocess All afterward will re-embed the remaining documents cleanly.
    • 
                    

                

                

                    📋 Live Logs — The last 200 log lines from the analyzer process. Auto-refreshes every 15 s.
                    
    
                        
  • INFO — Normal operation: documents found, analyzed, tags applied.
    • 
                        
  • WARNING — Non-fatal issues worth noting (rate limits, slow API responses, stale embeddings).
    • 
                        
  • ERROR — Failures: an analysis couldn't complete, an API call failed, a document couldn't be fetched.
    • 
                    

                    
If a document isn't being analyzed, search its title or ID here. Common causes: Paperless API 500s (check Health tab), LLM rate limits, or the document not matching any profile.
